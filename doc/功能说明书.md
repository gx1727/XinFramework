# XinFramework 功能说明书

## 1. 功能总览

XinFramework 当前以“后端基础框架 + SaaS 多租户骨架”为目标，功能分为三类：
- 基础能力：配置、启动、日志、错误恢复、统一响应
- 安全与隔离：JWT 鉴权、多租户上下文注入（TenantID）
- 领域预留：用户/角色/权限/租户、订阅与用量、AI 文档与向量

## 4. 中间件功能

### 4.1 日志与恢复

- Logger：使用 Gin 默认日志（[middleware.go](file:///d:/work/xin/XinFramework/internal/core/middleware/middleware.go#L57-L59)）
- Recovery：使用 Gin 默认恢复（[middleware.go](file:///d:/work/xin/XinFramework/internal/core/middleware/middleware.go#L61-L63)）

### 4.2 多租户（Tenant）

输入：
- Header：`X-Tenant-ID`

输出：
- 通过 `internal/core/context` 将 `TenantID` 写入上下文对象（[context.go](file:///d:/work/xin/XinFramework/internal/core/context/context.go#L7-L22)）

当前实现仅体现了“读取 Header 并尝试写入”的意图（[middleware.go](file:///d:/work/xin/XinFramework/internal/core/middleware/middleware.go#L43-L55)），租户隔离策略（shared/schema/database）尚未在 DB 层呈现。

### 4.3 鉴权（Auth）

输入：
- Header：`Authorization: Bearer <token>`

处理：
- 解析 JWT 并校验有效性（[middleware.go](file:///d:/work/xin/XinFramework/internal/core/middleware/middleware.go#L22-L31)）
- 从 claims 读取 `user_id` 并写入上下文（[middleware.go](file:///d:/work/xin/XinFramework/internal/core/middleware/middleware.go#L33-L38)）

输出：
- 失败：HTTP 401，返回 `{code:401,msg:"unauthorized|invalid token"}`（[middleware.go](file:///d:/work/xin/XinFramework/internal/core/middleware/middleware.go#L16-L30)）
- 成功：继续处理后续 handler

说明：
- 当前仅写入 `UserID`；`TenantID/Role` 未在中间件中消费
- JWT 生成函数在 `pkg/jwt` 中提供，但过期时间计算存在缺口（[jwt.go](file:///d:/work/xin/XinFramework/pkg/jwt/jwt.go#L15-L28)）

### 4.4 限流（预留）

`RateLimit()` 为占位实现，不做任何拦截（[middleware.go](file:///d:/work/xin/XinFramework/internal/core/middleware/middleware.go#L65-L69)）。

## 5. API 功能说明

### 5.1 统一响应结构

`pkg/resp` 定义了统一响应格式：
- 成功：`{code:0,msg:"ok",data:<payload>}`
- 失败：`{code:<业务码>,msg:<错误信息>,data:null}`
- 分页：`data` 内包含 `total/list`

参考：([resp.go](file:///d:/work/xin/XinFramework/pkg/resp/resp.go#L7-L26))

### 5.2 接口列表（当前仓库可见）

#### 5.2.1 健康检查

- 方法：GET
- 路径：`/api/v1/health`
- 鉴权：无
- 响应：`{code:0,msg:"ok",data:{status:"ok"}}`

参考：([register.go](file:///d:/work/xin/XinFramework/api/v1/register.go#L8-L14))

注意：`api/v1/register.go` 中的注册函数目前未在入口处调用，若要启用需在启动时显式注册路由。

#### 5.2.2 用户管理（占位路由）

这些路由在入口中被注册，但 handler 为空函数（无响应体），且被鉴权中间件保护（[main.go](file:///d:/work/xin/XinFramework/cmd/server/main.go#L34-L46)）：

| GET | `/api/v1/users` | 用户列表（待实现） | Bearer Token |

一致性说明（现状）：
- 部分表（如 permissions）无 `tenant_id` 字段，语义更偏“全局权限字典”
- 代码模型与 SQL 表并未完全一一对应（例如 user_roles/tenant_users 等未见对应模型）

## 7. 功能边界与约束（现状）

- 框架层仅提供骨架能力；业务能力需在 `internal/module/*` 中补齐 service/handler
- 多租户“模式切换”仅存在配置项，尚无运行时策略实现
- 当前仓库缺少可执行的登录/发 token API，调用受保护接口前需要自行生成 token 或补齐认证模块
