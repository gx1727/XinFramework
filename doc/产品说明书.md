## 2. 产品概述

XinFramework 旨在提供一套可复用的后端基础能力，支持多租户（SaaS）业务的通用诉求：统一配置、服务启动、鉴权、租户隔离、用户与权限模型、数据库与缓存接入、统一响应结构等。框架以 HTTP API 形式对外提供能力，并预留订阅计费、用量统计、AI 文档与向量检索等扩展方向。

### 2.1 产品定位

- 面向中小型团队：以较小的工程成本快速搭建可演进的 SaaS 后端骨架
- 面向平台型业务：以“租户”为核心组织单位，支持后续扩展到按 Schema/按 Database 隔离
- 面向工程化：提供一致的响应结构、可插拔中间件、基础设施初始化入口

### 2.2 目标与不做什么

产品目标（当前仓库已体现或已预留）：
- 提供稳定的 HTTP 服务骨架（Gin）
- 提供多租户上下文注入（TenantID）
- 提供 JWT 鉴权中间件（Auth）
- 提供用户/角色/权限/租户等核心表结构与模型
- 提供数据库（PostgreSQL）与缓存（Redis）初始化能力

不做什么（当前仓库未实现，需上层业务补齐）：
- 完整的业务 API（目前仅有健康检查与用户 CRUD 占位路由）
- 账号注册/登录/刷新令牌完整流程
- 完整的租户隔离（共享/Schema/数据库模式切换的运行时行为）
- 完整的审计、限流、指标监控与链路追踪

## 3. 目标用户与使用场景

### 3.1 目标用户

- 后端研发：以框架作为新项目起点，快速具备多租户与鉴权能力
- 测试工程师：基于统一接口规范与租户/鉴权规则制定测试用例
- 运维/平台工程：依据配置文件完成环境部署，接入数据库与缓存

### 3.2 典型场景

- 多租户管理后台：租户、用户、角色权限管理
- 订阅与用量计费：按租户订阅计划、记录用量并限制配额
- AI 文档中心：按租户存储文档并生成向量嵌入，支持检索增强

## 4. 核心概念

### 4.1 租户（Tenant）

- 租户是 SaaS 的一级隔离域，贯穿数据、权限与用量统计
- 当前数据结构中，核心表大多包含 `tenant_id`
- 请求侧通过 Header `X-Tenant-ID` 传入租户标识（中间件会尝试写入上下文）

相关实现参考：
- 多租户配置：`saas.mode`（[config.yaml](file:///d:/work/xin/XinFramework/configs/config.yaml#L26-L27)）
- Tenant 中间件：读取 `X-Tenant-ID`（[middleware.go](file:///d:/work/xin/XinFramework/internal/core/middleware/middleware.go#L43-L55)）

### 4.2 用户、角色、权限（User/Role/Permission）

- 用户：归属某个租户（`tenant_id`），具备状态、邮箱等属性（[model.go](file:///d:/work/xin/XinFramework/internal/module/user/model.go#L7-L21)）
- 角色：归属某个租户（`tenant_id`），用于权限集合（[model.go](file:///d:/work/xin/XinFramework/internal/module/user/model.go#L22-L29)）
- 权限：全局或跨租户定义（当前 SQL 未包含 `tenant_id`）（[001_init.sql](file:///d:/work/xin/XinFramework/migrations/001_init.sql#L23-L29)）

### 4.3 鉴权（JWT）

- 请求 Header `Authorization: Bearer <token>` 携带 JWT
- 中间件解析 JWT，并将 `user_id` 写入上下文（[middleware.go](file:///d:/work/xin/XinFramework/internal/core/middleware/middleware.go#L13-L41)）
- Claims 结构包含 `user_id/tenant_id/role`（[jwt.go](file:///d:/work/xin/XinFramework/pkg/jwt/jwt.go#L8-L13)）

### 4.4 订阅与用量（预留）

数据库迁移中包含 `subscriptions/plans/usage_records`（[001_init.sql](file:///d:/work/xin/XinFramework/migrations/001_init.sql#L59-L86)），用于支持：
- 租户订阅某个套餐计划
- 记录租户行为用量（action/count）
- 将配额与限流策略联动（当前仅预留 `RateLimit` 中间件占位）

### 4.5 AI 文档与向量（预留）

迁移包含 `ai_documents/ai_embeddings`，且 embedding 使用 `VECTOR(1536)`（[001_init.sql](file:///d:/work/xin/XinFramework/migrations/001_init.sql#L88-L104)），指向：
- 文档内容存储与检索
- 向量嵌入与相似度检索（依赖 PostgreSQL 的向量扩展能力）

### 5.2 已预留/未完成能力（需要补齐）

- 用户管理 API：当前仅在启动函数内注册了 CRUD 占位路由，处理逻辑为空（[main.go](file:///d:/work/xin/XinFramework/cmd/server/main.go#L34-L46)）

## 6. 非功能性要求（建议基线）

- 安全：JWT secret 不应使用默认值；生产环境必须安全管理密钥
- 可靠性：统一错误响应，避免 panic 泄露内部信息（Gin Recovery 已启用）
- 可观测性：建议扩展 request_id、结构化日志、指标与追踪
- 扩展性：按模块组织（`internal/module/*`），基础设施解耦（`internal/infra/*`）
