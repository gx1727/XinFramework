> 父文档：[0023-split-platform-tenant.md](0023-split-platform-tenant.md)。本文件是 0023.3 阶段的**详细施工单**——可执行步骤 + 验证清单 + 回滚预案。

## 1. 目标

| 项 | 现状 | 终态 |
|---|---|---|
| Go 包路径 | `apps/rbac/{user,role,menu,organization,permission,resource}` | `apps/tenant/{user,role,menu,organization,permission,resource}` |
| Framework 包 | `framework/pkg/rbac/{user,role,menu,organization,permission,resource}.go` | `framework/pkg/tenant/auth/{user,role,menu,organization,permission,resource}.go` |
| 表名（`tenant_*` 前缀） | `users / roles / organizations / user_roles / role_menus / role_resources / role_data_scopes / menus / resources` | `tenant_users / tenant_roles / tenant_organizations / tenant_user_roles / tenant_role_menus / tenant_role_resources / tenant_role_data_scopes / tenant_menus / tenant_permissions` |
| `apps/platform/menu` | 仍服务 `menus WHERE scope='platform'`（旧表） | 标记 Deprecated，业务迁到 `apps/platform/sys_menu` 后删除 |
| `tenant_id=1` 的 `bootstrap` 特殊租户 | 既是 admin 居住地、又是新租户克隆源 | admin 居住身份迁到 `sys_users`；bootstrap 保留作为克隆源 |

### 3.2 SQL 表重命名（任务 3.4 + 3.5 + 3.6）

> **风险标注**：大表重命名 = 锁表。生产环境必须在**维护窗口**执行；dev/staging 可随时。

#### 3.2.1 新建迁移文件 `migrations/0023_3_tenant_rename.sql`

```sql
-- ============================================
-- 0023.3 租户域表重命名
-- 依赖：0023.1 (双写 trigger 已关闭、account_roles 已 drop) + 0023.2 (sys_* 已 seed 并接管)
-- ============================================
-- ⚠️ 大表操作：分批 + 维护窗口；单事务包不住
-- ⚠️ RENAME TABLE 会改表名 + 索引名，**不改外键列名**

BEGIN;

-- 1. 先关掉所有 RLS policy（重命名后再开）
--    （policy 是按表名绑定的，rename 后旧 policy 自动失效，需重建）
-- 实际上 PG 行为：policy 会跟随表 rename；不需要显式 drop。下面只是注释作历史索引。

-- 2. 重命名（按依赖关系：先叶子，后根）
-- 2.1 关联表（叶子）
ALTER TABLE user_roles        RENAME TO tenant_user_roles;
ALTER TABLE role_menus        RENAME TO tenant_role_menus;
ALTER TABLE role_resources    RENAME TO tenant_role_resources;
ALTER TABLE role_data_scopes  RENAME TO tenant_role_data_scopes;

-- 2.2 主表
ALTER TABLE users        RENAME TO tenant_users;
ALTER TABLE roles        RENAME TO tenant_roles;
ALTER TABLE organizations RENAME TO tenant_organizations;

-- 2.3 业务表（待 0023.4 切流后再做；本阶段保留旧名 + 双读）
-- ALTER TABLE menus     RENAME TO tenant_menus;     -- 推迟到 0023.4
-- ALTER TABLE resources RENAME TO tenant_permissions; -- 推迟到 0023.6（与 RLS 改造一起）

-- 3. 重建 RLS policy（policy 跟随表 rename，但 policy name 不变——无需重建）
-- 实际上：PG 的 policy 内部用 oid 绑定表，rename 后 policy 自动跟随。无需手动重建。

-- 4. 索引名清理（可选）：旧索引名带原表名，建议显式重命名
ALTER INDEX IF EXISTS idx_user_roles_user   RENAME TO idx_tenant_user_roles_user;
ALTER INDEX IF EXISTS idx_user_roles_role   RENAME TO idx_tenant_user_roles_role;
ALTER INDEX IF EXISTS uk_user_roles         RENAME TO uk_tenant_user_roles;
-- ...（其余索引同理，按需 rename）

-- 5. 完整性校验
DO $$
DECLARE
    expected_tables TEXT[] := ARRAY[
        'tenant_users', 'tenant_roles', 'tenant_user_roles',
        'tenant_role_menus', 'tenant_role_resources', 'tenant_role_data_scopes',
        'tenant_organizations', 'tenant_user_seq'
    ];
    missing TEXT := '';
    t TEXT;
BEGIN
    FOREACH t IN ARRAY expected_tables LOOP
        IF NOT EXISTS (SELECT 1 FROM information_schema.tables
                       WHERE table_schema = 'public' AND table_name = t) THEN
            missing := missing || t || ', ';
        END IF;
    END LOOP;
    IF missing <> '' THEN
        RAISE EXCEPTION '0023.3 校验失败：缺失表 %', missing;
    END IF;
    RAISE NOTICE '0023.3 校验通过：% 张租户域表已重命名', array_length(expected_tables, 1);
END $$;

COMMIT;
```

# ⚠️ 警告：
#   1. 不能误伤 sys_users / sys_roles / sys_orgs / sys_user_roles / sys_role_menus / sys_role_permissions
#      —— 所以用 \b...\b 词边界，且替换目标不含 sys_ 前缀

## 4. 回滚预案

### 4.1 Go 包回滚（简单）

```bash
git revert <commit_sha>                          # 一次性 revert
# 或：
git mv apps/tenant apps/rbac
find . -name "*.go" -exec sed -i 's|gx1727.com/xin/apps/tenant/|gx1727.com/xin/apps/rbac/|g' {} +
```

### 4.2 SQL 表回滚（中等风险）

```sql
-- 在新连接中执行（避免污染原事务）
BEGIN;
ALTER TABLE tenant_user_roles      RENAME TO user_roles;
ALTER TABLE tenant_role_menus      RENAME TO role_menus;
ALTER TABLE tenant_role_resources  RENAME TO role_resources;
ALTER TABLE tenant_role_data_scopes RENAME TO role_data_scopes;
ALTER TABLE tenant_users           RENAME TO users;
ALTER TABLE tenant_roles           RENAME TO roles;
ALTER TABLE tenant_organizations   RENAME TO organizations;
COMMIT;
```

> ⚠️ **回滚窗口**：表重命名后 **24 小时内**可以低成本回滚；超过 24h 应用层已经写入新表名，回滚 = 数据双写风险。

### 4.3 紧急冻结

如果生产发现严重问题：
1. **代码回滚**（`git revert` + 重新部署）—— 立刻生效
2. **数据库不回滚**（避免双写混乱）—— 旧表 `users / roles / ...` 暂时不用，等下次维护窗口再处理
3. 记录在 [0023.3-incident-log.md](0023.3-incident-log.md)（如有）
