# Go, 开始

一切从一个问题开始：

> 为什么做一个 SaaS 系统，要引入那么多依赖？

这个问题困扰了我很久。每次启动新项目，都要面对：
- ORM 的学习曲线
- 配置文件的复杂性
- 过度抽象带来的调试困难

于是我决定：**做减法**。

---

## 简单的选择

Go 是我熟悉并喜欢的语言。它：
- 编译快速，部署简单
- 语法简洁，容易阅读
- 性能出色，并发友好

所以 xin 选择 Go 作为主力语言。

---

## 不要 ORM

大多数 Go 项目会用 GORM 或者其他 ORM 库。我选择不用。

原因很简单：**SQL 是强大的，不需要被隐藏。**

直接使用 `pgx/v5`，手写 SQL，可以：
- 完全掌控查询逻辑
- 轻松优化性能
- 调试时直接看到执行的 SQL

这不是复古，这是务实。
